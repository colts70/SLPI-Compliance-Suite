SFH/DFH is a minimal, standardized protocol that enables a website or domain owner to make a machine-verifiable, deterministic declaration of their root identity and purpose for the benefit of AI systems, search engines, and other consuming agents. It provides a simple, universal identity layer by placing a tiny, self-contained JSON-LD file at a well-known path (https://<domain>/.well-known/stack). RAG is a heuristic method that operates after a query, retrieving relevant (but unverified) content fragments and using them to condition a Large Language Model (LLM).
